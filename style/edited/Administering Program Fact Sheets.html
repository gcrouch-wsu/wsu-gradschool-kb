Start here: What you need to know

This page is for Graduate School Fact Sheet administrators — members of the Communications and Outreach team with the WordPress Administrator role. Administrators can edit every Fact Sheet field, including the restricted fields that are locked for program coordinators. Graduate program coordinators and directors do not use this page; their editing tasks are covered in Maintaining Program Fact Sheets.

This page covers the Fact Sheet name and web address, the Factsheet display name, Program Names and Degree Types, the Include in programs list setting, how those fields build the public Degree Programs landing page, adding Contributor accounts to the site, and giving a Contributor access to a Fact Sheet with the Editorial Access Manager.

Important: Name changes need approval first

Do not rename a Fact Sheet until the Office of the Provost and WSU Faculty Senate have approved the program or degree name change.

Renaming also affects links. The Fact Sheet title and its web address (permalink) are separate: editing the title does not change the web address, but editing the web address breaks existing links from graduate program websites and WSU's graduate application system, and slug changes are not redirected automatically. When a rename requires a new web address, coordinate the change with the sites that link to the Fact Sheet.

Before you begin

Navigation: Go to the Graduate School WordPress admin site, log in with your WSU NetID, and select Factsheets from the left menu.

Gather this information before you start:

  • The approved program or degree name, if you are renaming a Fact Sheet.
  • The WSU NetID of any user you are adding to the site.
  • The degree type and its classification (doctorate, master's, graduate certificate, and so on), if you are adding a new Degree Type.
  • Whether the program should appear in the public Graduate Degree Programs list.

Fact Sheet name and web address

The Fact Sheet name (the title field at the top of the edit screen) is the official program or degree name. It appears as the page heading on the public Fact Sheet and as the entry title in the Graduate Degree Programs list. The web address is generated from the title when the Fact Sheet is first created and does not change on its own afterward.

To rename a Fact Sheet after the change is approved:

  1. Open the Fact Sheet for editing and update the title.
  2. Leave the web address unchanged unless the rename requires a new one. To change it, select Edit next to the permalink below the title.
  3. If you change the web address, notify the program so its website links can be updated, and confirm the graduate application system links still work.
  4. Select Update.

Factsheet display name

The Factsheet display name is a short label for the Fact Sheet, entered at the top of the Factsheet Data section. On the Degree Programs landing page it becomes a sub-heading inside the program group, but only when it differs from the Program Name — when the two match, the degrees are listed directly under the program heading with no extra label.

Example: The Doctor of Philosophy in Chemistry and Master of Science in Chemistry Fact Sheets both use the display name Chemistry. Because that matches their Program Name, both degrees list directly under the Chemistry heading.

Use the display name to separate distinct tracks within one program — for example, a display name of Agriculture - Plant Health Management under the Agriculture program adds a labeled sub-group for those degrees. Keep it identical across every Fact Sheet that belongs in the same sub-group.

When the field is blank, the Fact Sheet title is used in its place. If no Program Name is assigned, the display name becomes the group heading itself.

Program Names and Degree Types

Two selection boxes on the right side of the edit screen categorize each Fact Sheet. Both are locked for contributors and appear greyed out to them.

Assign a Program Name

  1. In the Program Names box, check the program the degree belongs to.
  2. If the program is not in the list, select Add New Program name, enter the name, choose a parent program if one applies, and add it.
  3. Select Update to save.

The Program Name is the collapsible group heading on the Degree Programs landing page, and its first letter decides which A–Z section the program files under. Every Fact Sheet that shares a Program Name is collected under that one heading, so the name must match exactly across the program's Fact Sheets. Only the first Program Name checked is used.

You can manage the full list of Program Names from Factsheets > Program Names in the left menu.

Assign Degree Types

Degree Types drive the colored badges beside each degree on the Degree Programs landing page and determine which Filter by type selections show the degree. Assign one Degree Type for each degree the Fact Sheet represents (for example, Ph.D. and M.S.), plus any badge markers described below. A Fact Sheet with no Degree Type still appears in the list, but without a badge and only under All Programs.

Add a new Degree Type

  1. Go to Factsheets > Degree Types in the left menu.
  2. Enter the new degree type name and select Add New Degree Type.
  3. Open the new term for editing and set the Degree Classification.
  4. Select Update.

Always set the Degree Classification. It is what produces the badge and the type filter. Every doctoral term uses Doctorate; every master's term uses Master, whatever the degree is called.

Why some terms appear to work without a classification. When the classification is left on Other, the plugin falls back to the term's web address (slug) and produces a badge only if that slug happens to match a value in the table. A few long-standing terms match by coincidence — Doctorate, Master's, Professional Master's Degree, Graduate Certificate, and Administrator Credentials. Descriptive names do not: Master of Science has the slug master-of-science, which matches nothing, so it badges only because its classification is set to Master.

Do not rely on the slug. A term with no classification and a non-matching slug produces no badge and appears only under All Programs, with no error or warning anywhere. Setting the classification makes the term work regardless of what it is named.

Badge markers: Global Campus and 4+1 Master's Entry

Global Campus and 4+1 Master's Entry are not degree types. They are recorded as Degree Types on purpose, because the Degree Types box is what produces a badge and makes a degree filterable on the landing page. Add them alongside the degree's real type, never instead of it.

Example: A Master of Science offered online carries both Master of Science and Global Campus, so its row shows an M badge and a G badge and appears under both the Masters and Global Campus filters. A degree with only the Global Campus marker shows a G and nothing else.

Global Campus is the one term that depends on its web address. There is no Global Campus option in the Degree Classification list, so the badge works only while the term's slug stays exactly global-campus. Do not rename the term or edit its slug: the G badge and the Global Campus filter would stop working for every degree using it, with no error shown.

The 4+1 term is safer. Set its Degree Classification to Masters with 4+1 Entry and its badge no longer depends on the slug at all.

Include in programs list

The Include in programs list setting controls whether the Fact Sheet appears in the public Graduate Degree Programs list.

SettingEffect
YesThe Fact Sheet appears in the Graduate Degree Programs list and in the grad fair view.
NoThe Fact Sheet does not appear in the list, but it remains published and reachable at its direct URL — existing links from program websites still work.

Set this to No for programs that are being phased out, are not yet accepting students, or should be reachable only through direct links.

The Degree Programs landing page

The public Degree Programs page is an ordinary WordPress page whose content is a single shortcode:

[gsdegrees]

The plugin replaces that shortcode with the entire browsing interface — the search box, the Filter by type list, the A–Z rail, the program count, the expand and collapse controls, and every program group. Nothing in the list is typed by hand. Each entry is generated from published Fact Sheet data every time the page loads, so you never edit the landing page to change a program. You change the Fact Sheet, and the list updates.

Ordering is automatic. Programs sort alphabetically by Program Name. Within a program, degrees sort by classification — doctorate, master's, professional master's, 4+1, graduate certificate, Global Campus, credentials — and then by title.

Add the list to a page

  1. Create or edit the WordPress page that will hold the list.
  2. Add a Shortcode block and enter [gsdegrees].
  3. Publish or update the page.

The shortcode also accepts a grad fair variant, [gsdegrees gradfair="1"], which points every entry at the grad fair version of its Fact Sheet URL. Use it only on grad fair pages.

When a program looks wrong on the list

Every fix below is made on the Fact Sheet, not on the landing page.

  • "The program is missing from the list." Confirm the Fact Sheet is published and Include in programs list is set to Yes.
  • "A program's degrees are split into two groups." The Program Names do not match exactly. Assign the same Program Name term to each Fact Sheet.
  • "The program is filed under the wrong letter." The letter comes from the first character of the Program Name, not the Fact Sheet title.
  • "A degree has no badge, or is missing from a type filter." Its Degree Type is set to Other and its slug matches no classification. Set the Degree Classification on the term under Factsheets > Degree Types — Doctorate for doctoral degrees, Master for master's degrees.
  • "Global Campus badges disappeared across many programs." The Global Campus Degree Type term was renamed or its slug changed. Restore the slug to global-campus.
  • "An online degree shows only a G badge." The real degree type is missing. Global Campus is a marker, not a substitute — add the degree's own Degree Type as well.
  • "An unexpected sub-heading appears under the program." The Factsheet display name differs from the Program Name. Match them to remove the sub-heading.

Add a Contributor to the site

Graduate program coordinators and directors normally receive access automatically when they are assigned in myWSU Faculty Lists. Add a user manually when someone needs access outside that process.

  1. In the dashboard, go to Users > Add New.
  2. Enter the user's WSU NetID email address.
  3. Set the role to Contributor.
  4. Add the user.

A Contributor account alone does not grant editing access to any Fact Sheet. After adding the user, assign them to their Fact Sheet with the Editorial Access Manager, described next.

Give a user access to a Fact Sheet

The Editorial Access Manager box in the right sidebar of the Fact Sheet edit screen controls who can edit that Fact Sheet. Only administrators can change it; everyone else sees it greyed out. The box shows a status badge (for example, Open access or 2 users assigned) and three access modes:

  • Off: Open access — standard WordPress permissions apply.
  • Roles: Only users with the selected roles can edit.
  • Users: Only the selected individuals can edit.

To assign a coordinator to their Fact Sheet:

  1. Open the Fact Sheet for editing.
  2. In the Editorial Access Manager box, select the Users mode.
  3. On the Users tab, select the user. You can assign more than one user to the same Fact Sheet.
  4. Select Update. The status badge shows the number of users assigned.

How team access behaves:

  • When a Fact Sheet is first published with access set to Off, the plugin automatically switches it to Users mode with the Fact Sheet's author as the only assigned user.
  • Assigned users can edit the Fact Sheet but cannot delete it.
  • Administrators always keep full access regardless of the mode.
  • Assigned users who are Contributors still cannot edit restricted fields.
  • The Factsheets list screen has an Editorial Access Manager column showing each Fact Sheet's mode and assignments, so you can audit access without opening each one.
